Here's a question that seems so obvious it almost doesn't need asking: where are your memories?

The standard answer --- the one that has dominated cognitive science for over a century --- is that memories are stored somewhere in the brain. You experience something, it gets encoded, filed away, and later you retrieve it. Memory as warehouse. Memory as hard drive. Memory as library.

This metaphor is so deeply embedded in how we think about the mind that it's hard to see it as a metaphor at all. But it is one. And it has always struggled with what memory actually does.

## The Trouble with Storage

Consider what happens when you remember your tenth birthday. You don't experience a perfect replay. The memory is fragmentary, colored by your current mood, shaped by whatever prompted the recollection. If someone asks "was it sunny?" you might generate an answer --- and then believe it was always part of the memory. Ask again next year and the details might be different.

These aren't bugs. This is how memory works, always and for everyone:

- **Memories are reconstructive.** They change each time you recall them.
- **Memories are context-dependent.** What you remember depends on where you are, how you feel, what you were just thinking about.
- **Memories blend with imagination.** You can "remember" things that didn't happen, and you can imagine future events using the same neural machinery as memory.
- **Memory is creative.** People routinely fill in gaps with plausible details and then treat those details as genuine recollections.

The storage model handles all of this poorly. If memories are stored files, why do they change? If retrieval is lookup, why does context matter so much? Why is the line between remembering and imagining so blurry?

## An Alternative: Generation

Large language models suggest a different way to think about this. An LLM has no memory store. It doesn't save conversations, file away facts, or retrieve stored information. Yet it produces contextually appropriate continuations that *look like* memory. It can recall earlier parts of a conversation, maintain consistency over long stretches of text, and integrate information in sophisticated ways.

How? By generating each token from the preceding sequence, constrained by distributional patterns learned during training.

I propose that biological memory works the same way. What we call "remembering" is not retrieval from storage but *generative reconstruction* --- the brain generates a trajectory through experiential state-space, using whatever cues are available as the seed.

This explains everything that puzzles the storage model:

- Memories are reconstructive because they *are* constructions --- each recall is a new generation.
- Memories are context-dependent because context provides the seed for generation.
- Memory and imagination share neural machinery because they are the same process --- generation constrained by different inputs.
- Memory is creative because generation is inherently creative.

## What This Means

If memory is generation rather than storage, then the fundamental unit of cognition is not the representation but the *trajectory*. The mind doesn't store snapshots of the world. It learns the distributional structure of experience, and it uses that structure to generate contextually appropriate sequences of mental states.

This is the starting point for what I call the autoregressive brain framework --- and its implications extend far beyond memory, into language, consciousness, and the nature of the self.
